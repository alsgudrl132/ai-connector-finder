# AI Connector Finder 🔌

> **AI 커넥터 자동 식별 및 보관 시스템**
> 이름 모르는 커넥터를 카메라에 보여주면, AI가 종류를 인식하고 해당 커넥터가 담긴 슬롯을 캐러셀이 자동으로 회전시켜 사용자 앞까지 가져와 서보 도어를 열어주는 스마트 커넥터 보관함.

---

## 핵심 차별점

단순 버튼식 자판기가 아니다.

**사용자가 이름을 모르는 커넥터**를 카메라에 올리면, AI가 종류를 식별하고 해당 부품을 자동으로 찾아 꺼내주는 **AI 기반 전자부품 보조 시스템**이다.

역할을 명확히 분리한 것이 설계 포인트다.

- **AI (Raspberry Pi)** — 객체 인식/분류라는, OpenCV 룰 기반만으로는 풀기 어려운 문제 담당
- **MCU (STM32 + FreeRTOS)** — 모터·서보·상태 관리 등 실시간 기구 제어 담당
- **UART** — 두 시스템을 잇는 통신 경계

"인지(perception)"와 "제어(control)"를 물리적으로 다른 프로세서에 분리하고, 그 사이를 명시적 프로토콜로 연결한 구조.

## 시스템 아키텍처

```
        [ 사용자가 커넥터를 인식 공간에 올림 ]
                     │
        ┌────────────▼────────────┐
        │   Raspberry Pi 4 (인지)  │
        │  ─ Camera Module 3 (AF)  │
        │  ─ MobileNetV3 분류      │
        │  ─ 결과 문자열 생성      │
        └────────────┬────────────┘
                     │ UART (TX/RX, 115200)
                     │ "C:USB-C,98\n"
        ┌────────────▼────────────┐
        │  STM32F411RET6 (제어)    │
        │  ─ FreeRTOS 상태 머신    │
        │  ─ 캐러셀 스텝모터 제어  │
        │  ─ 서보 도어 제어        │
        │  ─ LCD / RGB LED / 버튼  │
        └────────────┬────────────┘
                     │
     ┌───────────────┼───────────────┐
     ▼               ▼               ▼
  캐러셀 회전     서보 도어       상태 표시
 (슬롯 정렬)     (개방/닫힘)    (LCD, RGB LED)
```

## 동작 흐름

| 단계 | 동작 | 상태 표시 |
|------|------|-----------|
| 1. 촬영 | 커넥터를 인식 공간에 올리고 `SCAN` 버튼 | LCD: `Scanning...` / LED: 노랑 |
| 2. AI 인식 | Pi가 이미지 분류 후 결과 전송 | LCD: `USB-C (98%)` |
| 3. 확인 | 사용자가 `OK` 버튼으로 확정 | LCD: `Press OK ?` |
| 4. 캐러셀 회전 | STM32가 해당 슬롯까지 스텝모터 회전 | LCD: `Moving...` |
| 5. 도어 개방 | 슬롯이 사용자 방향에 오면 서보가 도어 개방 | LCD: `Take Connector` / LED: 초록 |
| 6. 사용자 수령 | 사용자가 직접 커넥터를 꺼냄 | — |
| 7. 도어 닫힘 | 일정 시간(약 5초) 후 자동 닫힘 | LCD: `Thank You!` |
| 8. 대기 | 초기 상태로 복귀 | LCD: `Ready / Press SCAN` / LED: 파랑 |

> `CANCEL` 버튼은 어느 단계에서든 대기 상태로 복귀시킨다.

## 배출 방식 (설계 의도)

커넥터를 **자동으로 떨어뜨리지 않는다.** 원하는 슬롯이 사용자 앞에 오면 서보가 작은 도어만 열고, 사용자가 직접 꺼낸다.

- 걸림(jam) 없음
- 한 번에 두 개 이상 배출되는 사고 없음
- 구조 단순, 유지보수 쉬움

## 전원 구성 (⚡ 모두 5V 통합)

28BYJ-48(5V)과 SG90(4.8~6V)을 쓰므로 **별도 12V 레일이 필요 없다.** 전체를 단일 5V로 구동한다.

```
        5V 전원 (권장 5V / 5A 이상)
                 │
   ┌─────────┬───┴────┬─────────┬──────────┐
   ▼         ▼        ▼         ▼          ▼
Raspberry  ULN2003   SG90     LCD1602   STM32F411
  Pi 4    +28BYJ-48  서보      (I2C)    (5V→3.3V 내부 레귤)
(5V/3A↑)  (스텝모터)
   │                                        │
   └──────────── GND 공통 (모든 장치) ───────┘
```

**주의 사항**

- **모든 장치의 GND는 반드시 공통**으로 연결한다. (STM32 ↔ ULN2003 ↔ Pi ↔ 서보 GND 공통이 아니면 UART/모터 신호가 튄다)
- **서보 브라운아웃 주의:** SG90은 기동/정지 시 순간 전류가 크다(스톨 시 ~700mA). Pi와 서보가 같은 5V 레일을 공유하면 서보 동작 순간 Pi가 리셋될 수 있다. 대책 중 하나:
  - 서보 전원 라인에 대용량 전해 커패시터(예: 470~1000µF) 추가, 또는
  - Pi는 별도 5V/3A 어댑터로 분리 급전하고 GND만 공통
- 28BYJ-48은 저토크 부하(가벼운 캐러셀)에 적합. 캐러셀 무게가 늘면 NEMA 17 + A4988(12V) 조합으로 교체 검토.

## 소프트웨어

### Raspberry Pi 4 (인지)

- **언어/라이브러리:** Python, Picamera2(libcamera), TensorFlow Lite
- **모델:** MobileNetV3 (Image Classification)
  - 전이학습(Transfer Learning)으로 커넥터 클래스 학습 → `.tflite` 변환 → Pi에서 추론
- **판단 근거:** 인식 공간이 고정되고 한 번에 커넥터 하나만 올라오는 **통제된 환경**이므로, 바운딩 박스가 필요한 detection(YOLO 등)이 아니라 **classification**으로 충분. 학습 데이터·추론 비용이 낮고 Pi에서 안정적.
- **출력:** 분류 결과를 UART 문자열로 STM32에 전송

**학습 파이프라인**

```
커넥터 사진 수집        전이학습             모델 변환         Pi 추론
(클래스당 100~300장) → (MobileNetV3 기반) → (.tflite) → (실시간 분류)
  ※ 실제 캐러셀 조명/각도에서 촬영해야 실환경 정확도 확보
```

### STM32F411 (제어)

- **RTOS:** FreeRTOS
- **태스크 설계** (버튼/UART는 인터럽트로 이벤트화 → 큐로 상태 머신에 전달하는 구조)

| 태스크/ISR | 역할 |
|-----------|------|
| `UART_RX` (ISR/DMA) | Pi 수신 파싱 → 이벤트 큐 push |
| `Button` (EXTI ISR) | SCAN/OK/CANCEL 입력 → 이벤트 큐 push |
| `StateMachine` (Task) | 핵심 상태 전이, 큐 소비, 전체 흐름 제어 |
| `Motor` (Task) | 캐러셀 스텝모터 회전/슬롯 정렬 |
| `Servo` (Task) | 도어 개폐 + 자동 닫힘 타이머 |
| `Display` (Task) | LCD1602 / RGB LED PWM 갱신 |

> 버튼·LCD를 각각 독립 태스크로 두기보다, **인터럽트 → 큐 → 단일 상태 머신** 구조가 우선순위/경합 관리 면에서 깔끔하다.

- **캐러셀 인덱싱:** 전원 인가 시 리밋 스위치로 **홈 포지션 호밍** → 이후 오픈루프 스텝 카운트로 각 슬롯 위치 계산. 1회전 스텝 수 / 슬롯 수로 슬롯 간격 산출.
- **상태 머신:** `IDLE → SCANNING → CONFIRM → MOVING → OPEN → CLOSING → IDLE`

## UART 프로토콜

ASCII 라인 기반. 디버깅 편의를 위해 사람이 읽을 수 있는 형식 사용.

**Raspberry Pi → STM32** (분류 결과)

```
C:<class>,<confidence>\n     예) C:USB-C,98\n
```

**STM32 → Raspberry Pi** (상태 통보, 선택)

```
S:<state>\n                  예) S:MOVING\n , S:DONE\n
A:OK\n                       (수신 ACK)
```

- Baud rate: 115200, 8N1
- 프레이밍: `\n` 라인 구분. 필요 시 간단한 체크섬(XOR) 추가로 견고성 확보.

**클래스 예시** (10슬롯, 변경 가능)

```
USB-A, USB-C, Micro USB, Mini USB, HDMI,
RJ45, DC Jack, JST-XH, DisplayPort, Dupont(2.54mm)
```

## 기구 설계 (3D 프린팅)

- **원형 캐러셀** — 지름 약 180mm, 높이 약 60mm, 슬롯 10개(변경 가능)
- 각 슬롯에 커넥터 개별 보관, 스텝모터 1축으로 전체 회전
- 카메라 브래킷 / LCD 브래킷 / STM32·Pi 케이스
- 사용자 방향 서보 도어 + 서보 링크
- 스텝모터 마운트

**도어 메커니즘:** 서보가 도어를 위로 들어 올려 개방 → 사용자 수령 → 타이머 만료 시 자동 닫힘.

## 개발 단계 (권장 순서)

1. **STM32 단독** — 캐러셀 회전, 리밋 스위치 호밍, 서보 도어, LCD/LED/버튼 동작
2. **UART 테스트** — PC 시리얼에서 `C:USB-C,98` 수동 주입 → STM32가 해당 슬롯으로 회전하는지 검증
3. **AI 단독** — Pi에서 커넥터 3~4종 먼저 분류, 정확도 확인 후 클래스 확장
4. **통합** — Pi ↔ STM32 UART 연동, 전체 흐름 + 상태 머신 완성
5. **완성/확장** — 클래스 추가, 재고 관리, 로그 저장 등

> 3종만 돌아가도 데모 성립. 나머지는 "클래스 추가만 하면 됨"으로 확장성 어필 가능.
