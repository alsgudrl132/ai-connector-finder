# AI Connector Finder 🔌

> **AI 커넥터 자동 식별 및 보관 시스템**
> 이름 모르는 커넥터를 카메라에 보여주면 AI가 종류를 인식하고, 캐러셀이 슬롯을 하나씩 카메라 밑으로 돌려 같은 커넥터를 찾은 뒤, 그 자리에서 덮개를 열어 꺼내주는 스마트 보관함.

---

## 핵심 차별점

단순 버튼식 자판기가 아니다. **사용자가 이름을 모르는 커넥터**를 카메라에 보여주면, AI가 종류를 식별하고 캐러셀에서 같은 커넥터를 자동으로 찾아 꺼내주는 **AI 기반 전자부품 보조 시스템**이다.

설계의 두 가지 핵심 판단:

**1. 캐러셀은 스스로 뭘 들고 있는지 모른다 — 카메라가 실시간으로 읽는다.**
스텝 카운트로 "몇 번 슬롯에 뭐가 있다"를 미리 정해두는 방식이 아니라, 목표를 찾을 때까지 **캐러셀을 한 칸씩 돌리며 카메라가 각 슬롯을 직접 판독**한다. 그래서 슬롯에 어떤 커넥터를 어떤 순서로 넣든 상관없다. 인식(perception)이 위치 관리를 대신한다.

**2. 스캔 위치 = 배출 위치 = 한 지점.**
사용자가 모르는 샘플을 올리는 곳, 카메라가 보는 곳, 커넥터를 꺼내는 구멍이 **전부 같은 한 지점**이다. 카메라 하나로 샘플 스캔과 슬롯 순차 인식을 모두 처리한다.

역할 분리: **AI(Raspberry Pi)** 는 객체 인식/분류를, **MCU(STM32 + FreeRTOS)** 는 모터·서보·상태 관리 등 실시간 제어를 담당하고, **UART** 가 둘을 잇는다.

## 동작 흐름

전제: 상판 덮개가 캐러셀 전체를 덮고, 앞쪽에 **딱 한 칸 구멍**이 있다. 그 구멍 위에 카메라가 하향으로 붙어있고, 구멍의 슬라이드 덮개는 **랙&피니언으로 서보가 옆으로 여닫는다.**

| 단계         | 동작                                                          | 덮개     | 표시                                  |
| ------------ | ------------------------------------------------------------- | -------- | ------------------------------------- |
| 1. IDLE      | 대기                                                          | 닫힘     | LCD `Ready / Place sample` · LED 파랑 |
| 2. 샘플 스캔 | 구멍 위에 **모르는 샘플**을 올리고 SCAN                       | 닫힘     | `Scanning...` · LED 노랑              |
| 3. AI 판정   | Pi가 분류 → 결과 표시                                         | 닫힘     | `USB-A 97% / Remove & OK`             |
| 4. 개방      | 샘플 치우고 OK → 서보가 **덮개 옆으로 개방**                  | **열림** | `Searching...`                        |
| 5. 순차 인식 | **덮개 열린 채** 캐러셀이 한 칸씩 회전, 카메라가 각 슬롯 판독 | 열림     | `Reading... USB-A`                    |
| 6. 매칭      | 목표 발견 → **정지**                                          | 열림     | `Found: USB-A` · LED 초록             |
| 7. 수령      | 사용자가 커넥터를 꺼냄                                        | 열림     | `Take Connector`                      |
| 8. 닫힘      | 서보가 덮개 닫음 → IDLE 복귀                                  | 닫힘     | `Thank You!`                          |

> 5단계에서 덮개가 **열린 채로** 회전하는 게 핵심이다. 닫혀 있으면 카메라가 슬롯 안을 못 보기 때문.
> `CANCEL` 버튼은 어느 단계에서든 대기 상태로 복귀시킨다.

## 기구 구조 (하부 수납형)

코인 소터처럼 하부에 전자부품을 숨기는 **두꺼운 베이스** 구조.

```
        +-----------------+
        |   카메라 (하향)   |  <- 구멍 위, FFC 1m로 하부 라파까지
        +--------+--------+
   +-------------+-------------+
   |  상판 덮개 (1칸 구멍)        |  <- 슬라이드 덮개 + 랙&피니언(서보)
   +---------------------------+
   |  캐러셀 (10슬롯, 회전)       |  <- 28BYJ-48 D축
   +---------------------------+
   |  하부 수납 베이스           |
   |   Pi 4 · STM32 · 브레드보드 |
   +---------------------------+
        앞면: LCD · 버튼3 · LED
```

- **캐러셀** 지름 180mm, 슬롯 10개, 28BYJ-48로 회전
- **상판 덮개** 캐러셀 전체를 덮고 앞쪽 한 칸만 구멍 (스캔=배출 지점)
- **슬라이드 덮개** 구멍을 여닫음, **랙&피니언**으로 옆으로 개방
- **하부 베이스** 높이 약 55mm 수납칸에 Pi4 + STM32 + 브레드보드
- **앞면 패널** LCD1602 + 버튼 3(SCAN/OK/CANCEL) + RGB LED

### 덮개 개폐 = 랙&피니언 (필라멘트 프린팅)

서보 회전을 덮개 직선 이동으로 바꾸는 방식으로 **랙&피니언**을 선택했다.

- **피니언**(작은 기어): 서보 혼에 결합, 서보가 돌면 회전
- **랙**(톱니 막대): 슬라이드 덮개에 붙어, 피니언에 물려 직선 이동
- **가이드 레일**: 랙이 직선으로만 움직이게 잡아줌 (프린팅 홈)

**부품 조달**: 랙·피니언·레일 전부 **필라멘트로 프린팅**. 별도 기어·베어링·나사 구매 불필요. 유일한 비프린팅 부품은 **SG90에 기본 포함된 혼과 고정 나사**(피니언을 서보 축에 고정)뿐 — 추가 구매 없음.

> 트레이드오프: 프린팅 기어는 톱니 모듈/유격 튜닝이 필요하다. 매끄럽게 물리려면 몇 번 테스트 출력해서 clearance를 맞춰야 한다. 대신 구매 부품 없이 필라멘트만으로 완성된다.

## 전원 (전체 5V 통합)

28BYJ-48(5V)·SG90(5V) 모두 5V라 **12V 레일 불필요.** 단일 5V 구동.

```
  5V 전원 (권장 5V/5A 이상)
        |
   +----+------+------+------+-----+
   Pi4  ULN2003 SG90  LCD    LED   STM32
     +28BYJ-48  서보  1602        (5V->3.3V)
        |
        +-- GND 공통 (전 장치)
```

**주의:**

- 모든 GND 공통 연결 필수 (UART/모터 신호 안정화)
- **서보 브라운아웃 주의**: SG90 기동 시 순간 전류가 크다. Pi와 서보가 5V를 공유하면 Pi 리셋 가능 → 서보 전원에 대용량 커패시터(470~1000uF) 추가하거나, Pi는 별도 5V로 급전하고 GND만 공통.

## 소프트웨어

### Raspberry Pi 4 (인지)

- **라이브러리**: Python, Picamera2(libcamera), TensorFlow Lite
- **모델**: MobileNetV3 (Image Classification), 전이학습 -> `.tflite` -> Pi 추론
- **판단 근거**: 통제된 환경 + 한 번에 커넥터 하나 -> detection(YOLO) 대신 **classification**으로 충분. 학습·추론 비용 낮고 안정적.
- **순차 인식 로직**: 캐러셀 한 칸 회전 신호 -> 카메라 캡처 -> 분류 -> 목표와 비교 -> 일치하면 STOP, 아니면 다음 칸 요청. 이 판독 루프가 이 프로젝트의 핵심.

학습 파이프라인

```
커넥터 사진 수집         전이학습            변환        Pi 추론
(클래스당 100~300장) -> MobileNetV3 기반 -> .tflite -> 실시간 분류
  ※ 실제 구멍 조명/각도에서 촬영해야 실환경 정확도 확보
```

### STM32F411 (제어)

- **RTOS**: FreeRTOS
- **태스크 구조** (인터럽트 -> 큐 -> 단일 상태머신)

| 태스크/ISR      | 역할                                  |
| --------------- | ------------------------------------- |
| `UART_RX`       | Pi 명령 수신 파싱 -> 이벤트 큐        |
| `Button` (EXTI) | SCAN/OK/CANCEL -> 이벤트 큐           |
| `StateMachine`  | 전체 흐름 제어, 큐 소비               |
| `Motor`         | 캐러셀 스텝모터 (한 칸 회전 = 인덱싱) |
| `Servo`         | 랙&피니언 덮개 개폐 (PWM)             |
| `Display`       | LCD1602 / RGB LED                     |

- **캐러셀 인덱싱**: 전원 시 리밋 스위치로 홈 포지션 확립 -> 이후 한 칸씩 회전. 위치는 카메라 판독이 최종 확인.
- **상태머신**: `IDLE -> SCAN -> RESULT -> OPEN -> INDEX(loop) -> MATCH -> WAIT -> CLOSE -> IDLE`

## UART 프로토콜

ASCII 라인 기반 (디버깅 편의). 115200 8N1.

**Pi -> STM32** (명령)

```
STEP\n         캐러셀 한 칸 회전 (다음 슬롯 판독 준비)
OPEN\n         덮개 개방
CLOSE\n        덮개 닫힘
HOME\n         원점 복귀
```

**STM32 -> Pi** (상태)

```
S:HOMED\n / S:STEPPED\n / S:OPENED\n / S:CLOSED\n
BTN:SCAN\n / BTN:OK\n / BTN:CANCEL\n
```

> 순차 인식은 Pi 주도: Pi가 `STEP` 보내고 -> STM32 `S:STEPPED` 응답 -> Pi가 카메라 판독 -> 목표면 `OPEN`, 아니면 다시 `STEP`.

## 개발 단계

1. **STM32 단독** — 캐러셀 회전(홈+인덱싱), 랙&피니언 덮개 개폐, LCD/LED/버튼
2. **UART 테스트** — PC에서 `STEP/OPEN/CLOSE` 수동 주입 -> 동작 확인
3. **AI 단독** — Pi에서 커넥터 3~4종 분류, 순차 판독 루프 검증
4. **통합** — Pi <-> STM32 UART, 전체 흐름 완성
5. **확장** — 클래스 추가, 로그, 웹 대시보드

## 어필 포인트

- **AI가 위치 관리를 대신** — 스텝 카운트가 아니라 실시간 카메라 판독으로 목표 탐색 (슬롯 배치 자유)
- **스캔=배출 단일 지점** — 카메라 하나로 샘플·슬롯 모두 처리하는 간결한 구조
- **랙&피니언 전부 프린팅** — 구매 부품 없이 필라멘트 + 서보 기본 부속만으로 메커니즘 구현
- **AI <-> MCU 역할 분리** — UART로 경계를 명확히 나눈 시스템 설계
- **RTOS 실시간 제어** — 인터럽트/큐 기반 상태머신
